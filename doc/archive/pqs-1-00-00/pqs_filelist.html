Details of the files

The following is a brief breakdown of the headers comprising the type family.
It is not a substitute for documentation. I apologise for that, but I am working on providing further comprehensive descriptions of the workings of the type , so I hope this provides some insights

pq_types

The pq_types directory contains the headers required to implement the class, in shape of the quantities you require for use in programs.
You only need to #include the header(s) for the type(s) you use.
Or you can #include the all-types header - pq_types/pq_all_types.hpp
Note:If you require readymade output you will also need to #include util/utility_output.hpp, which provides basic stream output of value and units of the quantity. This is quite useful for testing

Where you cannot find the unit typedef you require, you can define it.
Obviously there are also many more physical quantities types than are listed here. I have only created the main mechanics ones. I aim to start on the electronics ones next. If you create a new one and wish it to be included I will be happy to add it, if its not already in( Please do check you are happy to implement the boost copyright requirements first) .

Units of dimension

Contains the definitions for units of dimension.
Adding units of dimension is slightly involved.
I will try to provide more explanations of the enum values, but basically they are the logbase10 values of the <enum>number_of_std_units_in() specialisations, return values rounded and multiplied by 1,000,000.
An excel2000 spreadsheet to calculate the values is included docs/log_mx_calc.xls.
Dont forget to proovide the relevant specialisations, which should be obvious from the header. As with everything else there are many more to be added.
